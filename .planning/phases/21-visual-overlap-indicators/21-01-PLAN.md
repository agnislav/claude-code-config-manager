  **Step 2: Create src/config/overlapResolver.ts**

  Create the overlap resolver module with these exports:

  1. `deepEqual(a: unknown, b: unknown): boolean` — Deterministic deep equality using JSON.stringify with sorted keys for objects. Direct `===` for primitives. Array order matters.

  2. `getOverlapColor(overlap: OverlapInfo): 'red' | 'green' | 'yellow' | 'none'` — Priority: isOverriddenBy||isDuplicatedBy='red', overrides='green', duplicates='yellow', else 'none'.

  3. `resolveSettingOverlap(key: string, currentScope: ConfigScope, allScopes: ScopedConfig[]): OverlapInfo` — For scalar settings. Reads `sc.config[key]`.

  4. `resolveEnvOverlap(envKey: string, currentScope: ConfigScope, allScopes: ScopedConfig[]): OverlapInfo` — Reads `sc.config.env?.[envKey]`.

  5. `resolvePluginOverlap(pluginId: string, currentScope: ConfigScope, allScopes: ScopedConfig[]): OverlapInfo` — Reads `sc.config.enabledPlugins?.[pluginId]`. The "value" is the boolean (enabled/disabled).

  6. `resolveMcpOverlap(serverName: string, currentScope: ConfigScope, allScopes: ScopedConfig[]): OverlapInfo` — Reads `sc.mcpConfig?.mcpServers?.[serverName]`. NOTE: MCP servers are in mcpConfig, not config.

  7. `resolveSandboxOverlap(sandboxKey: string, currentScope: ConfigScope, allScopes: ScopedConfig[]): OverlapInfo` — Handles nested keys like "network.allowedDomains" by splitting on '.'.

  8. `resolvePermissionOverlap(category: PermissionCategory, rule: string, currentScope: ConfigScope, allScopes: ScopedConfig[]): OverlapInfo & { overriddenByCategory?: string }` — Uses `rulesOverlap()` from utils/permissions.ts for glob matching. Two rules with same pattern but different categories are overrides (value = category string). Same category same pattern = duplicate. Include `overriddenByCategory` for tooltip rendering. Only checks higher-precedence scopes (the "isOverriddenBy" direction). Permission rules don't have a "lower precedence neighbor" since they're granular and per-scope — set `overrides`/`duplicates` to undefined for permissions.

  **Internal helper**: Create a private `resolveOverlapGeneric(currentScope, allScopes, getValueFn)` that implements the nearest-neighbor algorithm from RESEARCH.md. Each public function wraps this with the appropriate value getter. The permission resolver is special-cased since it uses glob matching instead of exact key equality.

  The algorithm for generic overlap:
  - Sort allScopes by SCOPE_PRECEDENCE index
  - Find nearest higher-precedence scope that has the entity (lower index in SCOPE_PRECEDENCE)
  - Find nearest lower-precedence scope that has the entity (higher index in SCOPE_PRECEDENCE)
  - Use deepEqual to determine override vs duplicate for each direction
