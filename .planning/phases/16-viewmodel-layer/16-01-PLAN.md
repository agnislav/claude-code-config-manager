Create the complete ViewModel type system and TreeViewModelBuilder that transforms ConfigStore data into display-ready descriptors for all tree node types.

Purpose: Decouple tree node display logic from VS Code TreeItem construction. The ViewModel layer becomes the single source of truth for what the tree displays, making the rendering pipeline testable and the nodes trivial mappers.

Output: Two new files in `src/viewmodel/` — `types.ts` (interfaces) and `builder.ts` (builder class). No existing files are modified.

<task type="auto">
  <name>Task 2: Implement TreeViewModelBuilder</name>
  <files>src/viewmodel/builder.ts</files>
  <action>
Create `src/viewmodel/builder.ts` containing a class-based `TreeViewModelBuilder` that mirrors the existing ScopeNode/SectionNode dispatch pattern but outputs ViewModels instead of TreeItems.

**Class structure:**
```
export class TreeViewModelBuilder {
  constructor(private readonly configStore: ConfigStore) {}
  build(sectionFilter?: ReadonlySet<SectionType>): BaseVM[]
}
```

**build() method:**
- If `configStore.isMultiRoot()`, call `buildMultiRoot(sectionFilter)` returning `WorkspaceFolderVM[]`
- Otherwise call `buildSingleRoot(sectionFilter)` returning `ScopeVM[]`

**buildSingleRoot(sectionFilter):**
- Get first workspace folder key from `configStore.getWorkspaceFolderKeys()`
- Get `allScopes` via `configStore.getAllScopes(key)`
- Filter out `ConfigScope.Managed` (matching existing behavior in configTreeProvider.ts)
- Apply scope lock: if `configStore.isScopeLocked(scope)` and not already readOnly, create effective ScopedConfig with `isReadOnly: true`
- Map each scope to `buildScopeVM(scopedConfig, allScopes, key, sectionFilter)`

**buildMultiRoot(sectionFilter):**
- Iterate `configStore.getWorkspaceFolderKeys()`
- For each key, get allScopes and discoveredPaths
- Create WorkspaceFolderVM with: kind=WorkspaceFolder, label=folderName, icon=ThemeIcon('root-folder'), collapsibleState=Expanded, contextValue='workspaceFolder', nodeContext with placeholder scope (User), keyPath=[], workspaceFolderUri=key
- Children = scope VMs (same as single root, per folder)

**buildScopeVM(scopedConfig, allScopes, workspaceFolderUri, sectionFilter):**
Mirror ScopeNode constructor exactly:
- label: `SCOPE_LABELS[scope]`
- icon: `ThemeIcon(SCOPE_ICONS[scope])`
- collapsibleState: Collapsed
- description: For project scopes with fileExists, use `vscode.workspace.asRelativePath(filePath, false)`. For non-project scopes, use `getShortPath(filePath)` (replace HOME with ~). If !fileExists: 'Not found'
- tooltip: `MarkdownString(SCOPE_DESCRIPTIONS[scope])`
- contextValue: Custom pattern `scope.{scopeName}.{editable|readOnly}[.missing]` (NOT the standard pattern — this matches ScopeNode.computeContextValue override)
- resourceUri: For User scope only, use lock URI scheme with locked/unlocked query
- nodeContext: scope, keyPath=[], isReadOnly, isOverridden=false, workspaceFolderUri, filePath
- id: computed via `computeId(nodeContext)` pattern from baseNode.ts
- children: section VMs (only if fileExists or Managed scope)

**buildSections(scopedConfig, allScopes, sectionFilter):**
Mirror ScopeNode.getChildren() section dispatch exactly. For each section type, check if data exists in config, check sectionFilter, then call the appropriate entity builder. Order: Permissions, Sandbox, Hooks, McpServers, Environment, Plugins, Settings.

Each SectionVM:
- label: `SECTION_LABELS[sectionType]`
- icon: `ThemeIcon(SECTION_ICONS[sectionType])`
- nodeType for contextValue: `section.{sectionType}` (matching sectionNode.ts)
- description: item count string (mirror SectionNode.getItemCount exactly)
- keyPath: `[jsonKey]` where jsonKey is 'enabledPlugins' for Plugins section, otherwise sectionType value
- children: entity VMs from the appropriate build method

**Entity builder private methods** (one per entity type). Each must:
1. Call the appropriate override resolver (except hooks which have no override)
2. Compute NodeContext with correct keyPath, isOverridden, overriddenByScope, filePath
3. Compute display state (label, description, icon with override dimming, contextValue, tooltip)
4. Compute id via `computeId(nodeContext)`
5. Compute click command for leaf nodes (collapsibleState===None with filePath and keyPath.length>0)
6. Apply override suffix to description when overridden: `"${description} (overridden by ${SCOPE_LABELS[overriddenByScope]})".trim()`
7. Build children eagerly (no lazy loading)

**Critical entity-specific behaviors (from RESEARCH.md catalog):**

- **Permissions**: buildPermissionGroups returns 3 PermissionGroupVMs (deny, ask, allow). Each group's children are PermissionRuleVMs. Rules use `resolvePermissionOverride` which returns `overriddenByCategory`. PermissionRule tooltip includes category-specific override message.

- **Settings**: Uses `resolveScalarOverride`. Expandable if value is non-null non-array object. Description is `formatValue(value)` for scalars, empty for objects. Object tooltip: JSON pretty-print in markdown code block. Icon: 'tools' with disabledForeground when overridden. Children: SettingKeyValueVMs for object settings.

- **SettingKeyValue**: icon 'symbol-field' with disabledForeground when overridden. keyPath: [parentKey, childKey]. Description: formatValue. Override inherited from parent key via resolveScalarOverride on parentKey.

- **Environment**: Uses `resolveEnvOverride`. Icon: 'terminal' with disabledForeground. Description: the value string. keyPath: ['env', envKey].

- **Plugins**: Uses `resolvePluginOverride`. Label: parsed from pluginId (split at second '@' for scoped packages like `@scope/name@version`). Description: version suffix. checkboxState: Checked/Unchecked based on enabled boolean. resourceUri: custom PLUGIN_URI_SCHEME with enabled/disabled query. keyPath: ['enabledPlugins', pluginId]. Custom tooltip: plugin description from `PluginMetadataService.getInstance().getDescription(pluginId)` plus override warning if overridden.

- **Sandbox**: Uses `resolveSandboxOverride`. Flattens network sub-properties (network.allowedDomains etc.). Description: `formatSandboxValue(value)` — arrays show `[N items]`, booleans show string, other primitives as-is. Icon: 'vm' with disabledForeground when overridden. keyPath: ['sandbox', ...key.split('.')]. Tooltip: bullet list for arrays.

- **MCP Servers**: No override resolution. Icon: 'plug'. keyPath: ['mcpServers', serverName]. Note: MCP config comes from `scopedConfig.mcpConfig.mcpServers`, NOT from `scopedConfig.config`. Uses mcpFilePath for filePath in nodeContext.

- **Hooks**: No override resolution. HookEventVM: icon 'zap', description: total hook count across matchers, keyPath: ['hooks', eventType]. HookEntryVM: icon based on type (command->'terminal', prompt->'comment-discussion', agent->'hubot'), label format: `[matcher] command/prompt/type`, keyPath: ['hooks', eventType, matcherIdx, hookIdx]. HookKeyValueVM: icon 'symbol-field', keyPath: ['hooks', eventType, matcherIdx, hookIdx, propertyKey].

**Shared helper functions** (private or module-level in builder.ts):
- `computeId(nodeContext: NodeContext): string` — replicate baseNode.computeId exactly
- `computeStandardContextValue(nodeType: string, isReadOnly: boolean, isOverridden: boolean): string` — standard `{nodeType}.{editable|readOnly}[.overridden]` pattern
- `computeCommand(collapsibleState, filePath, keyPath): vscode.Command | undefined` — leaf click command
- `computeOverrideTooltip(isOverridden, overriddenByScope): MarkdownString | undefined`
- `applyOverrideSuffix(description, isOverridden, overriddenByScope): string`
- `formatValue(value: unknown): string` — replicate from settingNode.ts (cannot import from tree/ to avoid viewmodel->tree dependency)
- `formatSandboxValue(value: unknown): string` — replicate from sandboxPropertyNode.ts
- `getShortPath(filePath: string | undefined): string` — replicate HOME replacement from scopeNode.ts

**Import requirements:**
- `vscode` for ThemeIcon, ThemeColor, TreeItemCollapsibleState, MarkdownString, Uri, TreeItemCheckboxState
- `ConfigStore` from `../config/configModel`
- All 5 override resolvers from `../config/overrideResolver`
- Types: ConfigScope, ScopedConfig, SectionType, HookEventType, PermissionCategory, NodeContext from `../types`
- Constants: SCOPE_LABELS, SCOPE_DESCRIPTIONS, SCOPE_ICONS, SECTION_LABELS, SECTION_ICONS, PERMISSION_CATEGORY_ICONS, PERMISSION_CATEGORY_LABELS, DEDICATED_SECTION_KEYS from `../constants`

- PLUGIN_URI_SCHEME from `../tree/nodes/pluginNode`
- PluginMetadataService from `../utils/pluginMetadata`
- All ViewModel types from `./types`

**Target size:** ~250-350 lines per user decision. Keep methods focused and DRY.
