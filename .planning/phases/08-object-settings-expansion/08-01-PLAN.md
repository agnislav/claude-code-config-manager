**1. Create `src/tree/nodes/settingKeyValueNode.ts`:**

A new leaf node representing a single key/value pair inside an object-type setting.

```typescript
class SettingKeyValueNode extends ConfigTreeNode {
  readonly nodeType = 'settingKeyValue';

  constructor(
    parentKey: string,
    childKey: string,
    value: unknown,
    scopedConfig: ScopedConfig,
    allScopes: ScopedConfig[],
  ) {
    // Inherit override status from parent setting key
    const override = resolveScalarOverride(parentKey, scopedConfig.scope, allScopes);
    const ctx: NodeContext = {
      scope: scopedConfig.scope,
      keyPath: [parentKey, childKey],
      isReadOnly: scopedConfig.isReadOnly,
      isOverridden: override.isOverridden,
      overriddenByScope: override.overriddenByScope,
      filePath: scopedConfig.filePath,
    };
    super(childKey, vscode.TreeItemCollapsibleState.None, ctx);
    this.iconPath = override.isOverridden
      ? new vscode.ThemeIcon('symbol-field', new vscode.ThemeColor('disabledForeground'))
      : new vscode.ThemeIcon('symbol-field');
    this.description = formatValue(value);  // reuse same formatValue from settingNode
    if (typeof value === 'object' && value !== null) {
      this.tooltip = new vscode.MarkdownString('```json\n' + JSON.stringify(value, null, 2) + '\n```');
    }
    this.finalize();
  }

  getChildren(): ConfigTreeNode[] { return []; }
}
```

Use `symbol-field` icon (represents a property/field) — this visually distinguishes child keys from parent `tools` icon.

Export `formatValue` from `settingNode.ts` so `settingKeyValueNode.ts` can import it. This cross-file dependency exists because both `SettingNode` (for scalar/array descriptions) and `SettingKeyValueNode` (for child value descriptions) need identical value formatting. Sharing the function ensures consistency — e.g., a nested object value within an expanded setting renders as `{N keys}`, an array value as `[N items]`, null as `null`, all matching the parent-level formatting. Add the `export` keyword to the existing `formatValue` function declaration.

**Important: SettingKeyValueNode is always a leaf (non-expandable).** Nested objects within an expanded object setting render as non-expandable leaf nodes with `{N keys}` description from `formatValue()`. Only the top-level object settings get expand arrows — this keeps the tree one level deep and avoids recursive complexity. Similarly, array values within an object render as `[N items]` and null values render as `null` — all via `formatValue()`.

**2. Modify `src/tree/nodes/settingNode.ts`:**

a) **Conditional collapsibility in constructor:**
   - Check if `value` is a non-null, non-array object: `typeof value === 'object' && value !== null && !Array.isArray(value)`
   - If true: use `TreeItemCollapsibleState.Collapsed`
   - If false: use `TreeItemCollapsibleState.None` (existing behavior)

b) **Replace `{N keys}` description with empty string for object settings:**
   - When the value is a non-null, non-array object, set `this.description` to `''` (empty string) instead of calling `formatValue(value)` which returns `{N keys}`. The expand arrow signals expandability; child nodes show the actual content. An empty string is preferred over `undefined` because `applyOverrideStyle()` appends to `this.description` — an empty string ensures the override suffix still works correctly.
   - Scalar settings keep their current description behavior (value shown inline via `formatValue`).
   - Array settings keep `[N items]` description unchanged.
   - **Note:** This means expandable object settings lose the click-to-reveal-in-file command. This is intentional — `baseNode.ts:applyClickCommand()` (line 60) checks `this.collapsibleState !== TreeItemCollapsibleState.None` and skips command assignment for expandable nodes. The click gesture is instead used for expand/collapse.

c) **Implement `getChildren()` for objects:**
   - If `this.value` is not a non-null, non-array object: return `[]` (unchanged)
   - Otherwise: iterate `Object.entries(this.value)` and return a `SettingKeyValueNode` for each entry, passing `this.key` as `parentKey` and entry key as `childKey`
   - Store `scopedConfig` and `allScopes` as private readonly fields on SettingNode (they are currently only used in constructor — they will now also be needed in `getChildren()`)

d) **Export `formatValue`:** Add `export` keyword to the existing `formatValue` function. This is required because `SettingKeyValueNode` imports it for consistent value rendering across both node types (see rationale in step 1 above).
