4. Register the `claudeConfig.collapseAll` command. This delegates to the VS Code built-in tree collapse command:

```typescript
const collapseAllCmd = vscode.commands.registerCommand('claudeConfig.collapseAll', () => {
  vscode.commands.executeCommand('workbench.actions.treeView.claudeConfigTree.collapseAll');
});
```

5. Register the `claudeConfig.expandAll` command. This must walk the entire tree recursively and reveal each deepest (leaf) node to force VS Code to expand all ancestors. The `treeView.reveal()` method expands the path from root to the revealed node:

```typescript
const expandAllCmd = vscode.commands.registerCommand('claudeConfig.expandAll', async () => {
  const leaves = collectLeaves(treeProvider);
  for (const leaf of leaves) {
    try {
      await treeView.reveal(leaf, { select: false, focus: false, expand: true });
    } catch {
      // Node may not be visible (e.g., filtered out); skip silently
    }
  }
});
```

6. Add a `collectLeaves` helper function (private in extension.ts, after the `runDiagnostics` function) that recursively walks the tree via `treeProvider.getChildren()` and collects all leaf nodes (nodes whose `getChildren()` returns an empty array) AND all expandable parent nodes. The strategy: reveal every node that has `collapsibleState !== None`, because `reveal` with `expand: true` expands that node:

```typescript
function collectExpandableNodes(treeProvider: ConfigTreeProvider): ConfigTreeNode[] {
  const result: ConfigTreeNode[] = [];
  const walk = (nodes: ConfigTreeNode[]) => {
    for (const node of nodes) {
      if (node.collapsibleState !== vscode.TreeItemCollapsibleState.None) {
        result.push(node);
        const children = treeProvider.getChildren(node);
        walk(children);
      }
    }
  };
  walk(treeProvider.getChildren());
  return result;
}
```

Then use in the command:
```typescript
const expandAllCmd = vscode.commands.registerCommand('claudeConfig.expandAll', async () => {
  const expandable = collectExpandableNodes(treeProvider);
  for (const node of expandable) {
    try {
      await treeView.reveal(node, { select: false, focus: false, expand: true });
    } catch {
      // Node may not be visible (e.g., filtered out); skip silently
    }
  }
});
```

7. Add `collapseAllCmd` and `expandAllCmd` to the `context.subscriptions.push(...)` call in the disposable registration section (step 10).

**Important:** The `treeProvider.getChildren()` method is public and already exposed for the tree data provider contract. The `collectExpandableNodes` function uses it without needing any new methods on `ConfigTreeProvider`.

**Why not `showCollapseAll: true` on `createTreeView`?** That built-in option adds a collapse button, but it renders at a fixed position (far right). We need custom positioning between lock and filter. The custom command delegates to the same internal VS Code collapse mechanism.
