## v0.4.0 Requirements

Requirements for this milestone. Each maps to roadmap phases.

### Lock UX Improvements

- [ ] **LOCK-11**: User scope is locked by default on extension activation (locked state set before first tree render)
- [ ] **LOCK-12**: Lock icons use state semantics — `$(lock)` shown when locked, `$(unlock)` shown when unlocked
- [ ] **LOCK-13**: Lock toggle button moves from User scope node inline to toolbar (view/title), positioned left of filter button

### Toolbar Controls

- [ ] **TOOL-01**: Collapse All button in toolbar collapses all expanded tree nodes to top-level
- [ ] **TOOL-02**: Expand All button in toolbar expands all tree nodes to full depth

### Object Settings Expansion

- [ ] **SETT-01**: Object-type settings (non-scalar values with nested keys) render as expandable tree nodes instead of `{N keys}` description
- [ ] **SETT-02**: Each key/value pair within an object setting appears as a child node with key as label and value as description

## Deferred Requirements

Tracked but not in current roadmap.

### EditValue Inline
- **EDIT-01**: Inline edit for scalar values (envVar, sandboxProperty, settings)
- **EDIT-02**: EditValue inline button re-enabled with `$(edit)` icon

### Filter Enhancements
- **FILT-V2-01**: Filter count displayed in `TreeView.description` (e.g., "3 sections") when filter is active
- **FILT-V2-02**: Filter state persisted across VS Code sessions via `workspaceState`

### Lock Enhancements
- **LOCK-V2-01**: Lock state persisted across VS Code sessions via `workspaceState`
- **LOCK-V2-02**: Lock extended to Project Shared and Project Local scopes (architecture supports via `Set<ConfigScope>`)

### Plugin Inline Buttons
- **PLUG-01**: Re-enable move/copy/delete inline buttons for plugins
