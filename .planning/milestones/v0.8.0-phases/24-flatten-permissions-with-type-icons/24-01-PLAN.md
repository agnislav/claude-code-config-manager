Purpose: Users see all permission rules in a single flat list with icons that immediately communicate each rule's type (allow/deny/ask), reducing nesting and improving scannability.
Output: Refactored builder producing flat PermissionRuleVM[] with category-specific icons, dead PermissionGroup code removed.
</objective>
