In `src/commands/pluginCommands.ts`, add a new exported function:

```typescript
export async function togglePluginEnabled(
  node: ConfigTreeNode,
  enabled: boolean,
  refreshTree?: () => void,
): Promise<void>
```

This function consolidates the shared logic from both the checkbox handler and the togglePlugin command:

1. Guard: check `node.nodeType !== 'plugin'` -> return
2. Extract `{ filePath, keyPath, isReadOnly }` from `node.nodeContext`
3. Call `guardReadOnly(node, MESSAGES.userScopeLocked)` — but the guard logic here is slightly different from the standard guardReadOnly: it only checks `isReadOnly` (not filePath absence), and if read-only, refreshes tree and returns. Check if the `guardReadOnly` helper from Plan 01 handles this, or inline the simple check: `if (isReadOnly || !filePath || keyPath.length < 2) { if (isReadOnly && scope === User) showInfo(userScopeLocked); refreshTree?.(); return; }`
4. Check `isWriteInFlight(filePath)` — if true, show `MESSAGES.writeInProgress` and return
5. Call `await withWriteRetry(filePath, () => setPluginEnabled(filePath, keyPath[1], enabled))` — if withWriteRetry catches, also call `refreshTree?.()`

In `src/extension.ts`:

Replace the `onDidChangeCheckboxState` handler body (lines 134-163) with:
```typescript
treeView.onDidChangeCheckboxState(async (e) => {
  for (const [item, state] of e.items) {
    const enabled = state === vscode.TreeItemCheckboxState.Checked;
    await togglePluginEnabled(item as ConfigTreeNode, enabled, () => treeProvider.refresh());
  }
});
```

Replace the `togglePlugin` command handler (lines 166-194) with:
```typescript
const togglePluginCmd = vscode.commands.registerCommand(
  'claudeConfig.togglePlugin',
  async (node?: ConfigTreeNode) => {
    if (!node || node.nodeType !== 'plugin') return;
    const currentEnabled = node.checkboxState === vscode.TreeItemCheckboxState.Checked;
    await togglePluginEnabled(node, !currentEnabled, () => treeProvider.refresh());
  },
);
```

Remove the direct imports of `setPluginEnabled` and `isWriteInFlight` from extension.ts since they are no longer used there (they are used inside pluginCommands.ts). Keep `showWriteError` import only if still needed; otherwise remove it too. Add `togglePluginEnabled` to the import from `./commands/pluginCommands`.
