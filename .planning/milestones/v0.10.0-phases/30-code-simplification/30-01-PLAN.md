<task type="auto">
  <name>Task 1: Create shared helper modules</name>
  <files>src/utils/commandHelpers.ts, src/utils/timestamp.ts, src/viewmodel/builder.ts</files>
  <action>
Create `src/utils/commandHelpers.ts` with these exported helpers:

1. **`withWriteRetry(filePath: string, action: () => void): Promise<void>`** — Wraps an action in try/catch, calling `showWriteError(filePath, error, action)` on failure. This replaces the pattern:
```ts
try { action(); } catch (error) { await showWriteError(filePath, error, () => { action(); }); }
```

2. **`guardReadOnly(node: ConfigTreeNode, message?: string): boolean`** — Returns `true` if the node is read-only (caller should `return`). Checks `isReadOnly || !filePath`. If `isReadOnly && scope === ConfigScope.User`, shows `MESSAGES.userScopeLocked` info message. Otherwise shows the provided message (defaulting to `MESSAGES.readOnlySetting`) as a warning. This replaces the 8 inline guard blocks across editCommands, deleteCommands, moveCommands, pluginCommands. Signature should accept the node and a custom message string for the non-User-locked case (since different commands use `readOnlySetting`, `readOnlyDelete`, `readOnlyMove`, `readOnlyCopy`).

3. **`pickEditableTargetScope(configStore: ConfigStore, sourceScope: ConfigScope, workspaceFolderUri: string | undefined, placeHolder: string, labelPrefix?: string): Promise<ScopedConfig | undefined>`** — Filters allScopes to exclude source scope, read-only, Managed, and locked scopes. Shows QuickPick with `SCOPE_LABELS`. Returns the selected ScopedConfig or undefined. The `labelPrefix` parameter defaults to empty (showing just the scope label) but can be set to `"Copy to "` or `"Move to "` for copy/move commands. This replaces the 5 nearly-identical scope-picker blocks in moveCommands and addCommands.

4. **`confirmOverwrite(itemName: string, scopeLabel: string): Promise<boolean>`** — Shows modal warning `Claude Config: "${itemName}" already exists in ${scopeLabel}. Overwrite?` with "Overwrite" button. Returns true if user confirmed. Replaces the 3 overwrite dialogs in moveCommands.

Create `src/utils/timestamp.ts` with:

5. **`formatTimestamp(): string`** — Returns `[HH:MM:SS.mmm]` formatted current time string. Replaces the 3 inline timestamp formatters in openFileCommands.ts (logRevealInFile), fileWatcher.ts, and configWriter.ts.

Then modify `src/viewmodel/builder.ts`:

6. **Merge `formatSandboxValue` into `formatValue`** — The two functions differ only in how they handle objects: `formatValue` returns `{N keys}` while `formatSandboxValue` returns `JSON.stringify(value)`. Add an optional `style` parameter: `formatValue(value: unknown, style?: 'summary' | 'raw')`. Default `'summary'` preserves current formatValue behavior. `'raw'` uses JSON.stringify for objects (current formatSandboxValue behavior). Delete the `formatSandboxValue` function. Update all 3 call sites of `formatSandboxValue` in builder.ts to use `formatValue(value, 'raw')`.
