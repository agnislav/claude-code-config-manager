**1. Create `src/tree/vmToNode.ts`** -- the central mapper function.

Export a single `vmToNode(vm: BaseVM): ConfigTreeNode` function that switches on `vm.kind` (NodeKind enum) to instantiate the correct node class. Import all 15 node classes (14 existing + WorkspaceFolderNode). Each case casts the BaseVM to its specific VM type and passes it to the node constructor.

Include a default case that throws `Error('Unknown NodeKind: ...')` for safety.

This file lives outside `nodes/` to avoid circular dependency issues -- nodes import from vmToNode.ts, and vmToNode.ts imports from node files. IMPORTANT: This creates a circular dependency since nodes call `vmToNode` in their `getChildren()` and vmToNode imports nodes. To break this cycle, DO NOT import vmToNode statically in node files. Instead, make `vmToNode` a static property on ConfigTreeNode base class, set by the provider at initialization. Nodes call `ConfigTreeNode.mapVM(child)` in their getChildren(). This keeps all imports unidirectional.

Pattern:
```typescript
// In baseNode.ts
export abstract class ConfigTreeNode extends vscode.TreeItem {
  static mapVM: (vm: BaseVM) => ConfigTreeNode;
  // ...
}

// In vmToNode.ts
export function vmToNode(vm: BaseVM): ConfigTreeNode { ... }

// In configTreeProvider.ts constructor
ConfigTreeNode.mapVM = vmToNode;
```

Then in parent node getChildren():
```typescript
getChildren(): ConfigTreeNode[] {
  return this.vm.children.map(ConfigTreeNode.mapVM);
}
```

**2. Simplify `src/tree/nodes/baseNode.ts`** -- remove finalize() and all compute methods.

Replace the current constructor signature `(label, collapsibleState, nodeContext)` with `(vm: BaseVM)`. The constructor:
- Calls `super(vm.label, vm.collapsibleState)`
- Sets `this.nodeContext = vm.nodeContext`
- Sets `this.id = vm.id`
- Sets `this.iconPath = vm.icon`
- Sets `this.description = vm.description`
- Sets `this.contextValue = vm.contextValue`
- Sets `this.tooltip = vm.tooltip`
- Conditionally sets `this.resourceUri`, `this.checkboxState`, `this.command` if present in VM

Remove entirely: `finalize()`, `computeId()`, `computeContextValue()`, `computeTooltip()`, `applyOverrideStyle()`, `applyClickCommand()`.

Remove the import of `SCOPE_LABELS` from constants (no longer needed).

Keep: `abstract readonly nodeType: string`, `abstract getChildren(): ConfigTreeNode[]`, `readonly nodeContext: NodeContext`.

Add: `static mapVM: (vm: BaseVM) => ConfigTreeNode` static property for the mapper pattern.

The base class does NOT store vm as a field. Base class just maps VM fields to TreeItem properties. Each subclass stores its own typed `private readonly vm: XxxVM` and passes it to `super(vm)`. The base constructor parameter is `vm: BaseVM`.
