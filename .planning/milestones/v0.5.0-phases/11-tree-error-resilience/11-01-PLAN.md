<objective>
Add error resilience to tree rendering and plugin checkbox operations so that malformed data never crashes the extension or leaves the UI in an incorrect state.

Purpose: Tree operations are user-facing and must handle errors gracefully. Unhandled exceptions in `getChildren()` or `findNodeByKeyPath()` crash the TreeView; failed plugin writes leave checkboxes in a state inconsistent with disk. This plan wraps all tree operations in try-catch guards and adds plugin checkbox rollback.

Output: Error-resilient tree rendering across all node types and self-correcting plugin checkbox state on write failures.
</objective>

    **1. `onDidChangeCheckboxState` handler (line ~123):**
    Current code catches the error and calls `showWriteError`. After `showWriteError`, add `treeProvider.refresh()` to revert the checkbox. The handler needs to become async since showWriteError is async:
    ```typescript
    treeView.onDidChangeCheckboxState(async (e) => {
      for (const [item, state] of e.items) {
        const node = item as ConfigTreeNode;
        if (node.nodeType !== 'plugin') continue;
        const { filePath, keyPath, isReadOnly } = node.nodeContext;
        if (isReadOnly || !filePath || keyPath.length < 2) continue;
        const enabled = state === vscode.TreeItemCheckboxState.Checked;
        try {
          setPluginEnabled(filePath, keyPath[1], enabled);
        } catch (error) {
          await showWriteError(filePath, error, () => {
            setPluginEnabled(filePath, keyPath[1], enabled);
          });
          treeProvider.refresh();
        }
      }
    });
    ```

    **2. `togglePlugin` command handler (line ~141):**
    Already has try-catch with showWriteError. Add `treeProvider.refresh()` after the showWriteError call:

    ```typescript
    } catch (error) {

      });
      treeProvider.refresh();
    }
    ```

    The `treeProvider.refresh()` rebuilds the tree from disk state, which effectively reverts the checkbox since the failed write didn't persist. The brief visual flash from the full refresh is acceptable per user decision.
