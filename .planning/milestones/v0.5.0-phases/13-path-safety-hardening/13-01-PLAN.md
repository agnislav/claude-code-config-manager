  <action>
Add input validation to the `claudeConfig.revealInFile` command handler in `openFileCommands.ts`:

1. **filePath validation:**
   - Check that `filePath` is a non-empty string (if not, return early silently — this handles internal wiring where no path is available)
   - Check that `filePath` does not contain `../` traversal segments — if it does, show error: `vscode.window.showErrorMessage('Cannot reveal file: path contains traversal sequences')` and log to output channel, then return
   - Build a list of known config paths by calling `getUserSettingsPath()` and `getManagedSettingsPath()` from `platform.ts`, and deriving project paths from `vscode.workspace.workspaceFolders` (using `path.join(folder.uri.fsPath, '.claude', 'settings.json')`, `path.join(folder.uri.fsPath, '.claude', 'settings.local.json')`, and `path.join(folder.uri.fsPath, '.mcp.json')`)
   - Validate that `filePath` is in the known config paths set. If not, show error: `vscode.window.showErrorMessage(\`Cannot reveal file: ${filePath} is not a known config path\`)` and log to output channel, then return
   - Keep the existing `fs.existsSync(filePath)` check after validation

2. **keyPath validation:**
   - Check `Array.isArray(keyPath)` — if not, show error and return
   - Check `keyPath.length > 0 && keyPath.length <= 10` — if empty, return silently; if over 10, show error: `vscode.window.showErrorMessage('Cannot reveal file: keyPath exceeds maximum depth of 10')` and return
   - Check every element is a string: `keyPath.every(k => typeof k === 'string')` — if not, show error and return

3. **Output channel logging:**
   - Import the output channel pattern from the extension. Since `openFileCommands.ts` registration function receives `context`, add an optional `outputChannel` parameter to `registerOpenFileCommands()`. The caller in `extension.ts` already creates the output channel — pass it through.
   - Log validation failures with format: `[HH:MM:SS.mmm] [revealInFile] rejected: {reason}` matching the existing log format convention from Phase 12.

4. **Import additions:**
   - Add imports for `getUserSettingsPath`, `getManagedSettingsPath` from `../utils/platform`
   - Add imports for `PROJECT_CLAUDE_DIR`, `PROJECT_SHARED_FILE`, `PROJECT_LOCAL_FILE`, `MCP_CONFIG_FILE` from `../constants`

Order of validation: (1) type checks on filePath/keyPath, (2) traversal check, (3) known-path whitelist, (4) fs.existsSync, (5) proceed with reveal.
